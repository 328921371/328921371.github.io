react ajax请求

发表于 2017-09-22 | 分类于 react

主要讲解了react的ajax请求 及react的路由功能

1. 首先先来看一下 react的ajax请求

react的ajax请求要 通过 ‘axios’ 模块

所以在node包中先安装 – npm install axios –save

之后再App.js中进行 导入 import axios from ‘axios’;

踩坑!!!

react中文文档讲的挺详细的,我讲一下我在学习过程中遇到的一些
问题

文档中的get请求挺好的,拿下来就可以用(数据请求写在挂载完成里)

componentWillMount() {
    var url = 'http://192.168.1.120:3000/get';
    var filters = {
        name: 'csing'
    }

    axios.get(url, {params: filters})
    .then(function(response) {
        console.log(response);
    })
    .catch(function(error) {
        console.log(error);
    });
}
但是!!!

直接使用post请求,会报错,而且参数传递不过去,于是上百度找找原因,找到了一个解决办法—创建一个参数对象

var data = new URLSearchParams(filters)

传递的参数改为data,不过作为一个程序员,不是在写bug,就是在写bug的路上,这句可以解决报错,但是还是不能传参数,
于是找到了下面的这种解决办法

导入qs解决post请求无参数问题

首先在node包进行安装—-npm install qs –save

然后在头部进行导入—-import qs from ‘qs’;

并且使用 qs.stringify() 对参数进行转化!!

axios.post('http://192.168.1.120:3000/post', qs.stringify(filters))
.then(function(response) {
    console.log(response);
})
.catch(function(error) {
    console.log(error);
});

关于 react 的axaj请求就到这里了

2. react的路由

三大主流框架之一的react路由到底有什么神奇之处呢,阮一峰老师的文档看完令我受益匪浅,下面我讲一下自己的感悟

首先是安装

第一种是 npm install react-router-dom –save

第二种是 npm install react-router –save

两个效果是一样的,至于区别其实就是,加了-dom是react关于dom操作的方法

安装好了后进入到App.js的页面,在头部进行导入

import { BrowserRouter as Router,Route,Link,Redirect } from ‘react-router-dom’;

  1. BrowserRouter 浏览器端路由对象
  2. as 改名,别名
  3. Route 路由的配置
  4. Link 连接
  5. Redirect 重定向

2.1 我先介绍一下最简单的路由使用功能

首先是 Router,它其实是一个容器,将整个div(#app)的包裹住,真正实现功能的是里面的Route

我在src的路径你创建了一个文件夹,并写了几个进行调整的页面,然后在App.js中进行导入组件

import Home from './components/Home';

<Router>
    <div className="App">
        <Link to='/home'>首页</Link> //Link to进行设置要跳转的地址    
    <Route path='/home' component={Home}></Route>// 配置一个路由,配置地址对应加载的组件
      </div>
  </Router>

2.2 如何设置默认跳转页面

Redirect提供了两种方法给我们

第一种使用单标签的形式

<Redirect from="/" to="/home" />

第二种使用双标签的形式

<Redirect to={{pathname:'/home',state:{from:'/'}}}></Redirect>

值得注意的是,这两种方法在控制台中都会爆出提醒,提醒编写者在篡改路由的跳转

2.3 如何在路由界面进行传递参数

同样也有两种方式

第一种是在url上进行传值

假设我要传递的是id

<Route path='/detail/:id' component={Detail}></Route>

跳转的时候就加上对应的参数,点击跳转后就可以将值传过去

<Link to={'/detail/' + id }> {name}</Link>

在跳转到的detail页面,可以获取到传递过来的id:   this.props.match.params.id

第二种:传递的时候将值一起传递过去,写在Lint to 中

<Link to={{pathname:url,state:{id:'666',name:'cs'}}}></Link>

在对应的页面进行获取 -- this.props.location.state(这里的state是你设置的对象名)
在this.props有许多路由的api,下面会介绍一些

2.4 react路由的this.props.history 操作

比较重要的几个:

1. 返回 -- this.props.history.goBack

2. 跳转 -- this.props.history.push('/goods') 括号里写跳转的链接
       -- 注意：push与replace的区别，一个是添加，一个是替换，历史记录中被替换的已经不存在了，所以浏览器回退不到替换前的页面。

2.5 react路由的嵌套

在路由中是可以嵌套路由的只要在加一个<Route>就可以

小知识:

<Route path={this.props.match.url} exact component={Goods}></Route>
exact 只有页面为 this.props.match.url 时,路由嵌套才会显示,所以点击后路径改变,路由消失
